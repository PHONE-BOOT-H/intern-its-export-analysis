# Project Spec: 개도국 교통·ITS 수출 경쟁 지형 분석

살아있는 스펙. 정해지는 대로 갱신한다. 미결은 `[정해야 함]`으로 표시한다.

## 문제

한국은 C-ITS/V2X를 개도국에 수출하려 한다. 그런데 "어디서 누구와 경쟁하는지"를 데이터로 보는 뷰가 없다. ITS Korea의 해외 발주 게시판은 공고를 나열할 뿐이고, 시장 리포트는 연간·범용이다. 공개 데이터로 경쟁 지형을 직접 그린다.

## 목표 사용자

1차는 나 자신(포트폴리오·데이터 실력). 부차적으로 이 분석을 보게 될 채용자, 그리고 브리핑에 참고할 수 있는 국제협력부.

## 산출물

1. 재현 가능한 파이프라인: 공개 데이터 → 정제 → 집계.
2. 경쟁 지형 대시보드: 국가별 낙찰 점유, 시계열, 부문 분해.
3. 짧은 전략 브리프: 한국이 어디서 이기고 지나, ITS는 왜 MDB에 안 보이나.

## MVP 범위

포함:
- [x] ADB 낙찰(국적·원산지)로 교통 경쟁 리그테이블
- [x] ITS 니치를 works/goods/consulting로 분해 (도로공사 vs 기술 구분)
- [ ] WB 낙찰 붙이기 (financesone DS01693)
- [ ] AidData 중국 교통 재원(1,665건) 층 붙이기
- [ ] 시계열·지역 분석, 대시보드, 브리프

제외:
- 나라별 "need 점수로 진출 타깃 고르기" (데이터로 기각됨, DECISION_LOG 참고)
- 라이브 자동갱신 도구 (포털 봇 차단으로 불가, 정적 분석으로 확정)
- 회사 내부·게이트 자료 사용

## 성공 기준

한 달 뒤 이 3개로 판정한다.
1. 공개 데이터에서 방어 가능한 경쟁 발견을 뽑았는가 (works vs 기술 분리, 양자 채널 함의 포함).
2. 처음부터 재현되는가 (data/raw → 스크립트 → 결과).
3. 과정이 커밋 히스토리로 남았는가.
